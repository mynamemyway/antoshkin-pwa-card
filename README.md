# Antoshkin Loyalty Card (PWA)

Система цифровых дисконтных карт для цветочного комбината [rassada1.ru](https://rassada1.ru/).

---

## 📱 О проекте

PWA (Progressive Web App) для автоматизации выдачи скидочных карт в розничных точках. Система позволяет клиентам получать виртуальные карты лояльности через смартфон без установки приложения из AppStore/Google Play.

### Преимущества

- **Для клиентов:**
  - Установка в один клик через браузер
  - Не занимает место в памяти телефона
  - Всегда с собой в смартфоне
  - Мгновенная регистрация через SMS

- **Для бизнеса:**
  - Исключение фейковых номеров
  - Единая база клиентов
  - Экспорт данных для маркетинга
  - Работа без интернета (офлайн-режим)

---

## 🔑 Ключевые функции

| Функция | Описание |
|---------|----------|
| **PWA** | Установка на смартфон без сторов, работает как нативное приложение |
| **SMS-верификация** | 4-значный код подтверждает номер телефона |
| **QR-код** | Генерируется в браузере, содержит данные карты |
| **Сессии** | Автоматический вход при повторном открытии (30 дней) |
| **Админ-панель** | Просмотр клиентов, экспорт в CSV |

## 📊 Сценарии использования (User Flow)

### Сценарий 1: Новый клиент (первый вход)

```
Клиент сканирует QR-код на кассе
    ↓
Открывается страница регистрации
    ↓
Вводит имя и телефон
    ↓
Получает SMS с 4-значным кодом
    ↓
Вводит код для подтверждения
    ↓
Виртуальная карта с QR-кодом готова
```

### Сценарий 2: Повторный вход (с активной сессией)

```
Клиент открывает приложение по иконке
    ↓
Система проверяет сессию (cookie)
    ↓
Сразу показывает карту (без форм и SMS)
```

### Сценарий 3: Вход с нового устройства

```
Клиент открывает приложение на новом телефоне
    ↓
Вводит номер телефона
    ↓
Получает SMS с кодом
    ↓
Подтверждает код
    ↓
Получает доступ к карте
```

### Сценарий 4: Незавершённая регистрация

```
Клиент начал регистрацию, но не ввёл код
    ↓
Возвращается в приложение позже
    ↓
Вводит телефон → получает новый SMS-код
    ↓
Подтверждает код → получает карту
```

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
# Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка окружения

```bash
# Копирование шаблона
cp .env.example .env

# Редактирование .env (при необходимости)
# SMS_API_KEY=your_api_key  # для продакшена
```

### 3. Запуск приложения

```bash
# Запуск сервера разработки
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Приложение доступно по адресу: `http://localhost:8000`

## 🧪 Тестирование

### Режим тестирования

По умолчанию включён тестовый режим SMS:
- **Код верификации:** `0000` (для любого номера)
- **SMS не отправляются** (код выводится в консоль)

Для отключения тестового режима в `app/services/sms_service.py`:
```python
TEST_MODE = False  # Использовать реальные SMS
```

## 🔐 Безопасность

| Мера | Описание |
|------|----------|
| **HttpOnly Cookie** | Токен сессии недоступен для JavaScript (защита от XSS) |
| **Secure Cookie** | Передача только по HTTPS |
| **SameSite=Lax** | Защита от CSRF-атак |
| **Срок сессии** | 30 дней, затем требуется повторная авторизация |
| **SMS-код** | 4 цифры, истекает через 5 минут |

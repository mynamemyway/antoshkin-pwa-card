# План интеграции SMS.ru API

## Цель
Реализовать отправку SMS через API SMS.ru вместо заглушки (stub mode) в соответствии с официальной документацией sms.ru.

---

## Текущее состояние

- ✅ В `.env` добавлен `SMS_API_KEY` (получен от SMS.ru: `cd531320-...`)
- ✅ В `app/config.py` есть поле `SMS_API_KEY`, `SMS_SENDER_NAME`, `DEBUG`
- ✅ В `app/services/sms_service.py` есть функция `send_sms()` с заглушкой
- ⚠️ `TEST_MODE = True` — жёстко задано в файле, все коды "0000", SMS не отправляются

---

## Требования

- Отправлять только 4-значный код верификации
- Без названия компании в тексте SMS
- Использовать существующую конфигурацию из `.env`
- Не создавать новых файлов
- Добавить `SMS_TEST_MODE` в `.env` для управления тестовым режимом SMS

  - `DEBUG=True` остаётся для отладки FastAPI
  - Разделить ответственность: `DEBUG` для приложения, `SMS_TEST_MODE` для SMS

### 1.2. Обновление .env.example
- **Цель:** Добавить пример `SMS_TEST_MODE` в шаблон
- **Файл:** `.env.example`
- **Логика:**
  - Добавить `SMS_TEST_MODE=True` в пример
  - Добавить комментарий о назначении переменной

### 1.3. Обновление config.py
- **Цель:** Добавить поле `SMS_TEST_MODE` в настройки
- **Файл:** `app/config.py`
- **Логика:**
  - Добавить поле `SMS_TEST_MODE: bool = True` в класс `Settings`
  - Загрузка из `.env` через pydantic-settings

## Этап 2: Слой Сервисной Логики

### 2.1. Обновление sms_service.py — импорты
- **Цель:** Добавить необходимые библиотеки для HTTP-запросов
- **Файл:** `app/services/sms_service.py`
- **Логика:**
  - Импортировать `requests` для вызова SMS.ru API
  - Импортировать `logging` для логирования

### 2.2. Обновление sms_service.py — константы
- **Цель:** Переключить управление тестовым режимом на `settings.SMS_TEST_MODE`
- **Файл:** `app/services/sms_service.py`
- **Логика:**
  - Удалить глобальную константу `TEST_MODE`
  - Использовать `settings.SMS_TEST_MODE` для определения режима
  - Сохранить константу `TEST_CODE = "0000"`

### 2.3. Обновление sms_service.py — generate_sms_code()
- **Цель:** Использовать `SMS_TEST_MODE` для генерации кода
- **Файл:** `app/services/sms_service.py`
- **Функция:** `generate_sms_code()`
- **Логика:**
  - Проверять `settings.SMS_TEST_MODE` вместо глобальной `TEST_MODE`
  - В тестовом режиме возвращать "0000"
  - В боевом режиме генерировать случайный 4-значный код

### 2.4. Обновление sms_service.py — send_sms()
- **Цель:** Реализовать отправку SMS через SMS.ru API
- **Файл:** `app/services/sms_service.py`
- **Функция:** `send_sms(phone: str, code: str)`
- **Логика:**
  1. Проверить `settings.SMS_TEST_MODE`:
     - Если `True` — вернуть заглушку (как сейчас)
  2. Если `False` — выполнить запрос к SMS.ru API:
     - Сформировать URL: `https://sms.ru/sms/send`
     - Подготовить параметры: `api_key`, `to`, `msg`, `json`
     - Отправить GET-запрос с таймаутом 10 секунд
     - Распарсить JSON-ответ
     - Проверить статус ответа (`OK` или ошибка)
  3. Обработать ошибки:
     - Ошибки сети (timeout, connection error)
     - Ошибки парсинга JSON
     - Ошибки API SMS.ru (неверный ключ, нет баланса)
  4. Вернуть кортеж `(success: bool, message: str)`

### 2.5. Обновление sms_service.py — логирование
- **Цель:** Добавить логирование для отладки
- **Файл:** `app/services/sms_service.py`
- **Логика:**
  - Создать logger в начале файла
  - Логировать успешную отправку SMS (тестовую и реальную)
  - Логировать ошибки отправки SMS

### 3.3. Тестирование обработки ошибок

- Не коммитить `.env` с реальным API ключом в git

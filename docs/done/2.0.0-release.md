# Релиз v2.0.0 - Авторизация по звонку (Check Call)

**Дата релиза:** 25.04.2026  
**Версия:** 2.0.0  
**Статус:** ✅ Production Ready

## 🚀 Главное изменение

Внедрена **авторизация через входящий звонок (Check Call)** как основной метод верификации пользователей. Теперь пользователям не нужно вводить код вручную — система автоматически подтверждает личность после входящего звонка от SMS.ru.

## 💰 Преимущества
- **Время:** Сокращено на 10-30 сек
- **Деньги:** Бесплатно для бизнеса и пользователей
- **Удобство:** Пользователям не нужно вводить код

---

## ✨ Новые возможности

### 1. Авторизация по звонку (Check Call)
- **Автоматическая верификация**: Пользователь получает звонок, система сама определяет факт ответа и подтверждает телефон.
- **Без ввода кода**: Никаких SMS с кодами и ручного ввода — всё происходит автоматически.
- **Поддержка polling**: Фронтенд опрашивает статус звонка каждые 2 секунды до подтверждения.
- **Webhook интеграция**: SMS.ru отправляет уведомление о статусе звонка на сервер.

### 2. Улучшенная работа с сессиями
- **Принудительная установка куки**: При успешной верификации через звонок кука `session_token` устанавливается всегда, даже если сессия уже существовала в БД.
- **Исправлена проблема потери сессии**: Раньше при повторной авторизации кука могла не установиться, теперь это исправлено.
- **PWA поддержка**: Приложение корректно работает как PWA — куки сохраняются между сессиями.

### 3. Умные редиректы
- **Клиентская навигация**: Если у пользователя есть валидная кука, фронтенд автоматически перенаправляет его на страницу карты.
- **Серверная защита**: При попытке доступа к `/card/{phone}` без сессии происходит редирект на главную.

## 🔧 Технические детали

### Backend изменения

#### Новые эндпоинты
```python
POST /api/auth/initiate          # Инициирование звонка
GET  /api/auth/check-call-status # Проверка статуса звонка (polling)
POST /api/auth/webhook/sms-ru    # Webhook от SMS.ru
```

#### Безопасность
- Флаг `secure=True` для production (HTTPS через nginx)
- Параметр `credentials: 'include'` во все fetch-запросы на фронтенде

### Frontend изменения

#### Verify страница (`templates/verify.html`)
- Реализован polling механизм для проверки статуса звонка
- Добавлена симуляция звонка для тестового режима (`CHECK_CALL_TEST_MODE=True`)
- Автоматический редирект на карту после успешной верификации

#### Base шаблон (`templates/base.html`)
- Клиентская проверка сессии при загрузке страницы
- Сохранение телефона в `localStorage` для PWA
- Умные редиректы на основе состояния авторизации

## 📦 Обновление до v2.0.0

### Требования
- Python 3.12+
- FastAPI 0.109+
- SMS.ru аккаунт с поддержкой Check Call
- HTTPS (nginx с SSL сертификатом)

### Шаги обновления

1. **Обновите код**
   ```bash
   git pull
   ```

2. **Проверьте переменные окружения**
   ```env
   AUTH_METHOD=check_call  # или sms, или both
   CHECK_CALL_TEST_MODE=False  # True для тестирования без реальных звонков
   SMS_API_KEY=your_api_id
   ```

3. **Перезапустите контейнеры**
   ```bash
   docker-compose down
   docker-compose up -d --build
   ```

4. **Проверьте работу webhook**
   Убедитесь, что SMS.ru может достучаться до вашего сервера:
   ```
   POST https://your-domain.com/api/auth/webhook/sms-ru
   ```

5. **Протестируйте авторизацию**
   - Зарегистрируйтесь через звонок
   - Проверьте, что кука устанавливается (DevTools → Application → Cookies)
   - Обновите страницу — редирект на карту должен работать

## 📝 Известные ограничения

1. **Требуется HTTPS в production**
   - Куки с флагом `secure=True` работают по HTTPS
   - Для локальной разработки можно использовать HTTP с `secure=False`

2. **Задержка webhook от SMS.ru**
   - Webhook может прийти с задержкой 1-3 секунд
   - Polling механизм компенсирует эту задержку

3. **Тестовый режим симулирует звонок**
   - В режиме `CHECK_CALL_TEST_MODE=True` звонок не совершается
   - Статус меняется автоматически через 3 секунды

# Баги и правки:

## Админ-панель

1. Требуется реализовать пагинацию (перелистывание страниц), когда верифицированных пользователей более 50ти - на проде их уже больше.
2. Поиск по номеру телефона работает только по показанным (последним) 50-ти пользователям - это некорректно. Администратор должен иметь возможность искать по всей базе номер телефона, а не только по последним 50.
3. Подсчёт верифицированных пользователей сейчас реализован по верифицированным только из 50-ти последних отображённых на странице, а должен показывать цифру всех верифицированных в базе.

---

## План исправления

### Задача 1: Пагинация в админ-панели

**Проблема:** Сейчас отображаются только последние 50 пользователей без возможности переключения страниц.

**Решение:**

1. **Backend (`app/api/routers.py`):**
   - Изменить эндпоинт `/admin` для приёма параметров `page` (номер страницы) и `per_page` (записей на страницу)
   - Рассчитать `offset = (page - 1) * per_page`
   - Передать в шаблон: список пользователей, номер текущей страницы, общее количество страниц, общее число пользователей

2. **Frontend (`templates/admin.html`):**
   - Добавить блок пагинации с кнопками «← Предыдущая», «1», «2», «3», «...», «Следующая →»
   - Кнопки должны быть активны/неактивны в зависимости от текущей страницы
   - Добавить query-параметры к ссылкам: `/admin?page=2`

**Файлы для изменения:**
- `app/api/routers.py` — функция `admin_panel()`
- `templates/admin.html` — добавить HTML-блок пагинации

---

### Задача 2: Поиск по всей базе (с сохранением мгновенного UX)

**Проблема:** JavaScript-фильтрация работает только на клиенте в пределах 50 отображённых записей.

**Решение:**

1. **Backend (`app/api/routers.py`):**
   - Добавить параметр `search` в эндпоинт `/admin`
   - При наличии `search` выполнять фильтрацию в SQLAlchemy: `User.phone.ilike(f"%{search}%")`
   - Возвращать отфильтрованный список пользователей (пагинация при поиске отключается — показываем все совпадения)

2. **Frontend (`templates/admin.html`):**
   - **Сохранить мгновенный поиск** (`oninput`), как сейчас — UX не меняется
   - Добавить **debounce 300-500мс** (задержка перед отправкой запроса, чтобы не спамить)
   - Заменить локальную JS-фильтрацию на **AJAX-запрос**: `fetch(/admin?search=${value})`
   - Получать с сервера полный HTML таблицы и заменять `innerHTML` контейнера
   - Добавить индикатор загрузки (спиннер) на время запроса

**Файлы для изменения:**
- `app/api/routers.py` — функция `admin_panel()`, добавить параметр `search`
- `templates/admin.html` — обновить JS-функцию `filterTable()` на AJAX-версию с debounce

**UX после изменений:**
- Администратор вводит номер → таблица обновляется мгновенно (как сейчас)
- Поиск идёт **по всей базе**, а не только среди 50 последних
- Визуально никаких изменений для пользователя

---

### Задача 3: Подсчёт всех верифицированных пользователей

**Проблема:** Сейчас считается только среди 50 последних: `{{ users|selectattr('is_verified')|list|length }}`

**Решение:**

1. **Backend (`app/api/routers.py`):**
   - Добавить отдельный SQL-запрос для подсчёта всех верифицированных: `db.query(User).filter(User.is_verified == True).count()`
   - Передать значение `verified_count` в шаблон

2. **Frontend (`templates/admin.html`):**
   - Заменить `{{ users|selectattr('is_verified')|list|length }}` на `{{ verified_count }}`

**Файлы для изменения:**
- `app/api/routers.py` — функция `admin_panel()`, добавить запрос `verified_count`
- `templates/admin.html` — использовать `{{ verified_count }}`

---

## Последовательность реализации

1. **Шаг 1:** Исправить подсчёт верифицированных (Задача 3) — простое изменение, не влияет на другие части
2. **Шаг 2:** Реализовать серверный поиск (Задача 2) — требует изменения backend + frontend
3. **Шаг 3:** Добавить пагинацию (Задача 1) — наиболее объёмное изменение, требует интеграции с поиском

## Технические детали

**Параметры пагинации:**
- `per_page = 50` (по умолчанию)
- `page = 1` (по умолчанию, минимальное значение)
- `offset = (page - 1) * per_page`

**Параметры поиска:**
- `search = ""` (по умолчанию, пустая строка = без фильтра)
- Поиск по полю `phone` через `LIKE '%search%'`

**URL после изменений:**
- `/admin` — первая страница, без поиска
- `/admin?page=3` — третья страница
- `/admin?search=%2B7999` — поиск по номеру (URL-encoded)
- `/admin?page=2&search=%2B7999` — поиск с пагинацией

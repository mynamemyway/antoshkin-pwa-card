# Инструкция по ручному тестированию

## 🚀 Запуск приложения

### 1. Активировать виртуальное окружение

```bash
# macOS/Linux
source venv/bin/activate
```

### 2. Запустить сервер

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 3. Открыть в браузере

```
http://localhost:8000/
```

---

## 🧪 Тестовый режим SMS

**По умолчанию включён тестовый режим!**

- **Код верификации:** `0000` (для любого номера)
- **SMS не отправляются** (код выводится в консоль)
- **Не требуется** SMS_API_KEY

## 🔧 Настройка .env (опционально)

Для локального тестирования **не требуется** изменение `.env`.

### Для продакшена (позже):

```env
# Database
DATABASE_URL=sqlite:///./loyalty.db

# SMS Service (SMS.ru)
SMS_API_KEY=your_real_api_key_here
SMS_SENDER_NAME=Antoshkin

# Отключить тестовый режим
# В app/services/sms_service.py изменить:
# TEST_MODE = False
```

### SECRET_KEY

Для локальной разработки подойдёт любое значение. Для продакшена: - ⚠️ работает без ключа локально, это нормально?

```bash
# Сгенерировать случайный ключ
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```

## 🐛 Возможные проблемы

### 401 Unauthorized на `/api/me`

**Нормально!** Означает, что нет активной сессии.

### 404 Not Found на `/favicon.ico`

Не критично. Можно добавить иконку позже. - ⚠️ в коде реализовано? куда добавть?

### Код "0000" не работает

Проверить в консоли:
```
[SMS STUB] For testing, use code: 0000 - ⚠️  нет такого лога
```

Если лога нет — проверить в `app/services/sms_service.py`:
```python
TEST_MODE = True  # Должно быть True
TEST_CODE = "0000"
```

### Страница не обновляется после верификации

Очистить кэш браузера (Cmd+Shift+R на macOS).

## 🔍 Анализ гипотезы

**Гипотеза:** Два пользователя одновременно регистрируются → оба запроса находят что пользователя нет → оба создают запись → один проходит верификацию, второй нет.

# План асинхронного рефакторинга Antoshkin Loyalty Card

## Введение

**Проблема:** В базе данных обнаружены пары пользователей с одинаковым `created_at`, где один верифицирован, второй — нет.

**Причина:** Приложение использует синхронные операции в асинхронных эндпоинтах:
- `db.query()` — синхронный blocking I/O
- `db.commit()` — синхронный blocking I/O  
- `requests.get()` в `send_sms()` — синхронный blocking I/O (до 10 секунд)

**Последствия:**
1. Блокировка event loop при запросах к SMS.ru
2. Race condition при одновременной регистрации
3. Блокировки SQLite при конкурентной записи

**Цель рефакторинга:** Сделать все I/O операции асинхронными, устранить race conditions, обеспечить атомарность отправки SMS.

## Этап 3: Слой Сервисной Логики

### 3.1. Асинхронные CRUD операции
*   **Цель:** Переписать CRUD функции как асинхронные
*   **Файл:** `app/services/crud.py`
*   **Функции / Сигнатуры:**
    *   `async def get_user_by_phone(db: AsyncSession, phone: str) -> Optional[User]`
    *   `async def create_user(db: AsyncSession, full_name: str, phone: str) -> User`
    *   `async def update_user(db: AsyncSession, user: User, update_data: dict) -> User`
*   **Логика:**
    1.  Заменить `db.query()` на `await db.execute(select(...))`
    2.  Заменить `db.commit()` на `await db.commit()`
    3.  Заменить `db.refresh()` на `await db.refresh()`
    4.  Использовать `result.scalar_one_or_none()` для получения результата

### 3.2. Асинхронная отправка SMS
*   **Цель:** Не блокировать event loop при запросе к SMS.ru
*   **Файл:** `app/services/sms_service.py`
*   **Функция / Сигнатура:** `async def send_sms_async(phone: str, code: str) -> Tuple[bool, str]`
*   **Логика:**
    1.  Заменить `requests.get()` на `async with httpx.AsyncClient()` + `await client.get()`
    2.  Сохранить логику тестового режима (возврат "0000")
    3.  Обработать исключения `httpx.TimeoutException`, `httpx.RequestError`
    4.  Вернуть кортеж `(bool, str)` как в синхронной версии

### 3.3. Обработка race condition при регистрации
*   **Цель:** Устранить дублирование пользователей
*   **Файл:** `app/api/routers.py` (эндпоинт `/api/register`)
*   **Логика:**
    1.  Обернуть вызов `create_user()` в `try/except IntegrityError`
    2.  При `IntegrityError`: выполнить `await db.rollback()`, затем повторный `get_user_by_phone()`
    3.  Вернуть существующего пользователя

### 3.4. Атомарная отправка SMS
*   **Цель:** Не сохранять код если SMS не отправлена
*   **Файл:** `app/api/routers.py` (эндпоинты `/api/send-sms`, `/api/login`)
*   **Логика:**
    1.  Сгенерировать код
    2.  Вызвать `await send_sms_async()` ДО записи в БД
    3.  Если SMS не отправлена: выбросить `HTTPException(500)` без коммита
    4.  Если SMS отправлена: записать код, выполнить `await db.commit()`

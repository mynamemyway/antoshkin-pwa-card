### 7.2. Таймер как работает?
- Таймер жизни проверочного кода из СМС заканчивается и сам начинает заново отсчёт? Или есть сценарий по которому он останавливается раньше времени? Не активная вкладка браузера?
- Таймер должен просто остановиться на 00.00 когда время закончится - теперь так и происходит - ок.
- Кнопка "Отправить повторно" работает корректно - тут всё хорошо.
- Закончилось время 5 минут, ввёл код и он не сработал - всё верно
- Закончилось время 5 минут, сразу ввёл код и он сработал - почему? Предполагаю, что он не умирает для уже зарегистрированного пользователя? Проверить.
- При отправке повторного СМС выскакивает некрасивое всплывающее системное уведомление - убрать, оно избыточно.
*В целом таймер работает хорошо, пока не понимаю баги разовые или есть ошибки в логике*

**Ответы по пунктам:**

1. **Таймер жизни кода:**
   - Код хранится в БД с полем `sms_code_expires_at` (datetime)
   - Таймер на фронтенде просто показывает оставшееся время
   - **Не перезапускается автоматически** — нужно нажать "Отправить повторно"

2. **Таймер остановился на 00:00:**
   - ✅ Так и задумано — код больше не действителен

3. **Код сработал после истечения времени:**
   - **Баг!** Проверка в роутере должна блокировать вход с истёкшим кодом.
   - Возможно, для уже верифицированного пользователя (`user.is_verified=True`) проверка пропускается — это нужно исправить.

4. **Системное уведомление (alert):**
   - **Баг!** В `verify.html` есть `alert('Новый код отправлен!')`
   - Нужно заменить на красивое UI-сообщение.
   
**Исправление alert:**
- Заменить системный alert на скрытый блок в HTML, который показывается на 3 секунды после отправки кода.
- Код сработал после истечения времени для `user.is_verified=True` (если пользователь вышел и входит повторно должна быть проверка по смс) — исправить.

### 7.4. Сделать возможность тестировать на телефоне
*Прокинуть через ngrok? Есть другие альтернативы для РФ кроме ngrok?*

**Ответ:** Да, нужно для тестирования PWA на реальном устройстве.

**Варианты реализации:**

**Вариант A: ngrok (международный)**
```bash
ngrok http 8000
# Получаешь https://xxxx-xxxx.ngrok.io
```
- ✅ Просто, быстро
- ❌ Могут быть проблемы с оплатой из РФ
- ❌ Ограничения на бесплатном тарифе

**Вариант B: Cloudflare Tunnel (работает в РФ)**
```bash
# Установить cloudflared
brew install cloudflared

# Запустить туннель
cloudflared tunnel --url http://localhost:8000
```
- ✅ Бесплатно
- ✅ Работает в РФ
- ✅ HTTPS из коробки
- ✅ Нет ограничений по времени

**Вариант C: Локальная сеть (только для тестирования)**
```bash
# Запустить с доступом из локальной сети
uvicorn app.main:app --host 0.0.0.0 --port 8000

# Узнать свой IP
ipconfig getifaddr en0  # macOS
# или
ifconfig | grep inet    # Linux

# На телефоне открыть: http://192.168.x.x:8000
```
- ✅ Не нужен внешний доступ
- ❌ Только в одной Wi-Fi сети
- ❌ Нет HTTPS (PWA не будет работать)

### 7.6. Улучшение PWA
- Добавление иконок для iOS (apple-touch-icon)
- Splash screen для нативного вида при запуске

**Статус:** Опционально, можно добавить позже.

**Варианты реализации:**

**1. Apple Touch Icon**
```html
<!-- templates/base.html -->
<link rel="apple-touch-icon" href="/static/icon-180.png">
```
- Нужно создать PNG 180x180
- ✅ Быстро

1. Создать иконку:
  - Формат: PNG
  - Размер: 180x180 пикселей (для iOS)
  - Можно также 192x192 и 512x512 для Android/PWA

2. Положить в папку:
  - static/icon-180.png
  - static/icon-192.png
  - static/icon-512.png

3. Обновить manifest.json:
```
    {
      "icons": [
        {
          "src": "/static/icon-192.png",
          "sizes": "192x192",
          "type": "image/png"
        },
        {
          "src": "/static/icon-512.png",
          "sizes": "512x512",
          "type": "image/png"
        }
      ]
    }
```
4. Обновить base.html:
<link rel="apple-touch-icon" href="/static/icon-180.png">

Итого: Ты можешь нарисовать иконку в любом графическом редакторе (Photoshop, Figma, Canva) → сохранить как PNG → положить в static/ → обновить manifest.json и base.html. Никакой дополнительный код не нужен!

**2. Splash Screen**
```json
// static/manifest.json
{
  "splash_pages": ["/splash.html"]
}
```
- Требует отдельный HTML
- ⏳ Средне

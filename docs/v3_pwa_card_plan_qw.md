### 7.4. Сделать возможность тестировать на телефоне
*Прокинуть через ngrok? Есть другие альтернативы для РФ кроме ngrok?*

**Ответ:** Да, нужно для тестирования PWA на реальном устройстве.

**Варианты реализации:**

**Вариант A: ngrok (международный)**
```bash
ngrok http 8000
# Получаешь https://xxxx-xxxx.ngrok.io
```
- ✅ Просто, быстро
- ❌ Могут быть проблемы с оплатой из РФ
- ❌ Ограничения на бесплатном тарифе

**Вариант B: Cloudflare Tunnel (работает в РФ)**
```bash
# Установить cloudflared
brew install cloudflared

# Запустить туннель
cloudflared tunnel --url http://localhost:8000
```
- ✅ Бесплатно
- ✅ Работает в РФ
- ✅ HTTPS из коробки
- ✅ Нет ограничений по времени

**Вариант C: Локальная сеть (только для тестирования)**
```bash
# Запустить с доступом из локальной сети
uvicorn app.main:app --host 0.0.0.0 --port 8000

# Узнать свой IP
ipconfig getifaddr en0  # macOS
# или
ifconfig | grep inet    # Linux

# На телефоне открыть: http://192.168.x.x:8000
```
- ✅ Не нужен внешний доступ
- ❌ Только в одной Wi-Fi сети
- ❌ Нет HTTPS (PWA не будет работать)

### 7.6. Установка ярлыка / инструкция
*Куда пропала инструкция по установке ярлыка на рабочий экран смартфона (андроид /айфлон)?*
- Она была выполнена отлично, но сейчас я её не вижу, почему?
- Ты говорил, что на андроид приложение устанавливается автоматически, но я не вижу этого - оно само не установилось. Нужно либо реализовать это, либо сделать инструкцию также как для айфона.
- Она была сделана в красивой синей обводке, но сейчас я не вижу. Проверь, не удалил ли ты её случайно?
- И как реализовать чтобы инструкция показывалась только тем пользователям, у которых ярлык ещё не установлен?

**Ответ:** Инструкция не удалена, она скрыта (`class="hidden"`). PWA не устанавливается автоматически — всегда требуется действие пользователя.

**Пользовательский флоу:**

**Android:**
1. Первый заход → синяя инструкция (5 сек) → [×] скрыть
2. Пользователь: ⋮ → Установить приложение → иконка на главном экране
3. После 2+ посещений (если не установил) → Chrome баннер внизу
4. Повторный заход → инструкция не показывается (localStorage)

**iOS:**
1. Первый заход → синяя инструкция (5 сек) → [×] скрыть
2. Пользователь: Поделиться → На экран Домой → иконка на главном экране
3. Повторный заход → инструкция не показывается (localStorage)

**План реализации:**

**1. Вернуть инструкцию (синяя обводка)**
- Файл: `templates/card.html`
- Блок: `iosHint` (переименовать в `installHint`)
- Стиль: `bg-blue-50 border border-blue-200 rounded-lg p-4`

**2. Универсальная инструкция для iOS и Android**
```
┌─────────────────────────────────────────────┐
│  ⚠️  Установите приложение                  │
│                                             │
│  iPhone:                                    │
│  Нажмите «Поделиться» → «На экран Домой»    │
│                                             │
│  Android:                                   │
│  Нажмите «⋮» → «Установить приложение»      │
│                                             │
│  [×] Не показывать снова                    │
└─────────────────────────────────────────────┘
```

**3. Логика показа (JavaScript)**
- Показывать только если НЕ standalone режим
- Сохранять в localStorage `install_hint_dismissed: true` при закрытии
- Показывать на 5 секунд при первом заходе
- Скрывать после установки (matchMedia)

**4. Детали реализации**
- Файл: `templates/card.html`
- Функция: `showInstallHint()` в `showCard()`
- События: `beforeinstallprompt` (Android), `matchMedia` (iOS)
- Хранение: `localStorage.setItem('install_hint_dismissed', 'true')`

**5. Требования**
- ✅ Показывать только тем, у кого не установлено
- ✅ Универсальный текст для iOS/Android
- ✅ Возможность скрыть ("Не показывать снова")
- ✅ Автоскрытие через 5 секунд
- ✅ Синяя обводка (как было)

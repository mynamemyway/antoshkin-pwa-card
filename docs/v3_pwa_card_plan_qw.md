### 7.4. Сделать возможность тестировать на телефоне
*Прокинуть через ngrok? Есть другие альтернативы для РФ кроме ngrok?*

**Ответ:** Да, нужно для тестирования PWA на реальном устройстве.

**Варианты реализации:**

**Вариант A: ngrok (международный)**
```bash
ngrok http 8000
# Получаешь https://xxxx-xxxx.ngrok.io
```
- ✅ Просто, быстро
- ❌ Могут быть проблемы с оплатой из РФ
- ❌ Ограничения на бесплатном тарифе

**Вариант B: Cloudflare Tunnel (работает в РФ)**
```bash
# Установить cloudflared
brew install cloudflared

# Запустить туннель
cloudflared tunnel --url http://localhost:8000
```
- ✅ Бесплатно
- ✅ Работает в РФ
- ✅ HTTPS из коробки
- ✅ Нет ограничений по времени

**Вариант C: Локальная сеть (только для тестирования)**
```bash
# Запустить с доступом из локальной сети
uvicorn app.main:app --host 0.0.0.0 --port 8000

# Узнать свой IP
ipconfig getifaddr en0  # macOS
# или
ifconfig | grep inet    # Linux

# На телефоне открыть: http://192.168.x.x:8000
```
- ✅ Не нужен внешний доступ
- ❌ Только в одной Wi-Fi сети
- ❌ Нет HTTPS (PWA не будет работать)

### 7.5. Дизайн карты
*Дизайн карты сделать в стиле и размере apple wallet:*
- Вверху надписи оставить также как есть (название карты и магазина)
- Ниже вставить изображение скидочной карты по горизонтали (макет есть .tif)
- Ниже под изображением карты вставить наш QR
- В самом низу также оставить надписи как есть сейчас
*Итого: вставить макет скидочной карты над QR, сохранив все функциональные и ознакомительные надписи*

**Ответ:** Нужно изменить вёрстку `card.html`.

**Варианты реализации:**

**Структура макета:**
```
┌─────────────────────────────────┐
│   Антошкин дворик               │  ← Заголовок
│   Карта лояльности              │
├─────────────────────────────────┤
│                                 │
│   [ИЗОБРАЖЕНИЕ КАРТЫ .tif]      │  ← Макет карты
│   (горизонтальное, ~400x250)    │
│                                 │
├─────────────────────────────────┤
│   [QR-код] мини № карты: #1     │  ← QR-код
│   Увеличение по клику (180x180) │
├─────────────────────────────────┤
│   Владелец: Иван Иванов         │  ← Данные
│   Телефон: +7 (999) 123-45-67   │
└─────────────────────────────────┘
```

**Изменения в card.html:**
```html
<!-- Добавить изображение карты -->
<div class="text-center mb-6">
    <img src="/static/card-mockup.tif" 
         alt="Скидочная карта" 
         class="w-full h-auto rounded-lg shadow-md">
</div>

<!-- QR-код ниже -->
<div class="flex justify-center mb-6">
    <div id="qrcode" class="bg-white p-4 rounded-lg shadow-lg"></div>
</div>
```

**Требования:**
- Конвертировать `.tif` в `.png` или `.svg` (веб-форматы)
- Оптимизировать размер (<100KB)
- Добавить в `static/`

### Этап 2: Слой API (Схемы и Роутеры)

#### 2.1. Перевод ошибок API на русский (7.3)
* **Цель:** Заменить английские сообщения об ошибках на русские.
* **Файл:** `app/api/routers.py`
* **Логика:**
  - `POST /api/verify`:
    - "SMS code expired" → "Срок действия кода истёк"
    - "Invalid SMS code" → "Неверный код"
    - "No SMS code sent" → "Код не был отправлен"
    - "User not found" → "Пользователь не найден"
  - `POST /api/send-sms`:
    - "Failed to send SMS" → "Не удалось отправить SMS"
  - `POST /api/register`:
    - "Phone already registered" → "Телефон уже зарегистрирован"

#### 2.2. Исправление проверки для верифицированных пользователей (7.2)
* **Цель:** Исправить баг, когда код срабатывает после истечения времени для уже верифицированных пользователей, которые разлогинились и пытаются залогиниться повторно.
* **Файл:** `app/api/routers.py`
* **Логика:**
  - В `POST /api/verify` для пользователя с `is_verified=True`:
    - Не создавать сессию автоматически
    - Требовать ввод SMS-кода как для обычного пользователя
    - Проверять `sms_code_expires_at` перед созданием сессии
